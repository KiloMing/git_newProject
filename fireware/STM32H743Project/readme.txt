# 主控简介：STM32H743IIT6

## 1. 主控概述

本项目采用 **STM32H743IIT6** 作为系统主控制器。STM32H743IIT6 是意法半导体推出的 STM32H7 系列高性能微控制器，内置一颗最高主频可达 **480 MHz** 的 Arm Cortex-M7 内核，支持 DSP 指令集、双精度浮点运算单元（DP-FPU）以及一级指令缓存和数据缓存，适合运行实时操作系统、图形界面、通信协议栈和多传感器数据处理程序。

本项目使用的主控型号为：

| 项目 | 参数 |
|---|---|
| 主控型号 | STM32H743IIT6 |
| CPU 内核 | Arm Cortex-M7 |
| 最高主频 | 480 MHz |
| Flash | 2 MB |
| 内部 RAM | 约 1 MB |
| 封装 | LQFP176 |
| 工作电压 | 1.62 V～3.6 V |
| 浮点运算 | 单精度、双精度 FPU |
| 指令扩展 | DSP 指令、Thumb-2 指令集 |

STM32H743IIT6 具有较强的运算能力和丰富的片上外设，可满足化工厂有害气体监测系统对实时性、稳定性、图形显示、无线通信和多任务处理的要求。

---

## 2. 在本项目中的作用

STM32H743IIT6 是整个有害气体监测系统的核心控制单元，主要负责以下任务：

1. 采集各类有害气体传感器和环境传感器的数据。
2. 对传感器原始数据进行滤波、标定和浓度换算。
3. 运行 RT-Thread 实时操作系统，完成多任务调度。
4. 驱动 LCD，并通过 LVGL 显示气体浓度、温湿度、设备状态和报警信息。
5. 通过 LoRa 与远程检测节点进行无线通信。
6. 通过 ESP8266 或其他网络模块将监测数据上传至云平台。
7. 控制蜂鸣器、报警灯、继电器和排风设备。
8. 保存历史监测数据和系统运行记录。
9. 检测传感器故障、通信故障和系统异常。
10. 在气体浓度超过安全阈值时执行本地声光报警和远程告警。

STM32H743IIT6 在系统中既承担底层硬件驱动任务，也承担数据处理、图形显示、网络通信和设备管理任务。

---

## 3. 主要硬件资源

### 3.1 高性能处理器

STM32H743IIT6 采用 Arm Cortex-M7 内核，最高运行频率为 480 MHz，并集成以下计算加速资源：

- 单精度和双精度浮点运算单元；
- DSP 数字信号处理指令；
- 16 KB 指令缓存；
- 16 KB 数据缓存；
- 紧耦合存储器 TCM；
- ART Accelerator 指令加速单元。

这些资源可以提高传感器数据处理、数字滤波、界面刷新和通信协议解析的执行效率。

### 3.2 存储资源

STM32H743IIT6 内置 2 MB Flash 和约 1 MB RAM，可用于保存程序代码、RT-Thread 内核、LVGL 图形界面资源、通信缓冲区和传感器数据。

当内部 RAM 无法满足 LCD 显存和大型图形资源需求时，可通过 FMC 外设扩展 SDRAM。项目所使用的正点原子阿波罗开发板带有外部 SDRAM，可用于存放 LVGL 显存、图片和动态数据。

### 3.3 模拟采集资源

STM32H743IIT6 集成多组高分辨率 ADC，可用于采集模拟气体传感器、电池电压和其他模拟信号。

在本项目中，ADC 可配合 DMA 工作，实现：

- 多通道循环采样；
- 定时触发采样；
- 低 CPU 占用的数据搬运；
- 多传感器连续监测；
- 软件滤波和浓度计算。

### 3.4 通信接口

STM32H743IIT6 提供丰富的通信外设，包括：

- USART/UART；
- SPI；
- I2C；
- FDCAN；
- USB OTG FS/HS；
- Ethernet MAC；
- SDMMC；
- Quad-SPI；
- SAI 和 I2S。

在本项目中，可以按照以下方式分配通信接口：

| 接口 | 连接设备 | 主要用途 |
|---|---|---|
| UART | ESP8266 | 云平台通信 |
| UART | LoRa 模块 | 远程节点数据传输 |
| UART | 调试串口 | 日志输出和参数调试 |
| I2C | 温湿度或环境传感器 | 数字传感器数据采集 |
| SPI | 外部 Flash 或无线模块 | 高速数据通信 |
| SDMMC | SD 卡 | 历史数据和日志存储 |
| Ethernet | 网络接口 | 有线网络通信扩展 |
| USB | 上位机 | 调试、升级或数据导出 |

### 3.5 显示与外部存储接口

STM32H743IIT6 内置 LTDC 液晶显示控制器和 FMC 外部存储器控制器，适合驱动彩色 TFT-LCD 和外部 SDRAM。

在本项目中：

- LTDC 或开发板对应的 LCD 接口负责图像输出；
- FMC 负责连接 SDRAM；
- SDRAM 用于存放 LVGL 显存；
- LVGL 负责绘制仪表盘、曲线、报警窗口和设备状态界面。

## 4. 选择 STM32H743IIT6 的原因

### 4.1 满足 RT-Thread 多任务运行需求

有害气体监测系统需要同时执行传感器采集、界面刷新、LoRa 通信、云平台上传、报警控制和数据存储等任务。

STM32H743IIT6 的处理性能和存储资源能够支持 RT-Thread 将不同功能划分为独立线程，提高系统的软件结构清晰度和实时响应能力。

### 4.2 满足 LVGL 图形界面需求

LVGL 图形界面需要较大的 RAM、较高的处理速度和稳定的 LCD 刷新能力。STM32H743IIT6 具备高主频、缓存、LTDC、DMA2D 和 FMC 等资源，适合构建具有以下内容的人机交互界面：

- 实时气体浓度；
- 温湿度信息；
- 历史变化曲线；
- 设备在线状态；
- 报警级别；
- 网络连接状态；
- 参数设置界面。

### 4.3 支持多种通信方式

化工厂监测系统通常需要同时支持现场总线、短距离通信、远距离无线通信和互联网通信。STM32H743IIT6 提供多组 UART、SPI、I2C、CAN、USB 和 Ethernet 接口，可以连接多种通信模块，并为后期扩展预留接口。

### 4.4 具备较好的扩展能力

STM32H743IIT6 具有较多 GPIO 和外设资源，可扩展更多气体传感器、报警装置、继电器、存储器和通信模块。

与原项目使用的 STM32F429 相比，STM32H743IIT6 在处理性能、缓存、存储容量和外设带宽方面更强，适合作为项目升级后的主控制器。

## 5. 系统结构

```mermaid
flowchart LR
    A[气体传感器] --> H[STM32H743IIT6]
    B[温湿度传感器] --> H
    C[其他环境传感器] --> H

    H --> D[LCD + LVGL界面]
    H --> E[蜂鸣器与报警灯]
    H --> F[继电器与排风设备]
    H --> G[SD卡或外部Flash]

    H <--> L[LoRa模块]
    H <--> W[ESP8266网络模块]
    W <--> P[云平台]
    L <--> N[远程监测节点]
```

STM32H743IIT6 位于系统中心，对下连接传感器和执行机构，对上连接本地显示界面、无线通信模块和云平台。

---

## 6. 软件运行结构

项目计划采用 RT-Thread 实时操作系统，并将主要功能划分为多个线程：

| 线程 | 主要功能 |
|---|---|
| Sensor Thread | 周期性读取气体和环境传感器 |
| Data Process Thread | 滤波、标定和浓度计算 |
| LVGL Thread | 图形界面刷新和用户交互 |
| LoRa Thread | 接收远程节点数据并发送控制信息 |
| Wi-Fi Thread | 连接云平台并上传监测数据 |
| Alarm Thread | 判断报警等级并控制报警设备 |
| Storage Thread | 保存历史数据和运行日志 |
| System Monitor Thread | 检测任务、内存和通信状态 |

这种设计可以降低不同功能之间的耦合度，使程序更容易调试、维护和扩展。

## 7. 开发环境

本项目可使用以下软件和工具进行开发：

- STM32CubeMX：配置时钟、GPIO 和片上外设；
- STM32CubeIDE 或 Keil MDK：编写、编译和调试程序；
- STM32 HAL/LL：完成底层外设驱动；
- RT-Thread：实现实时任务调度和设备管理；
- LVGL：实现本地图形化人机界面；
- ST-Link：完成程序下载和在线调试；
- Git：进行源代码版本管理。

当前项目基于正点原子 STM32H743 阿波罗开发板进行开发，可在已有 HAL 例程和板级支持代码的基础上逐步完成 RT-Thread、LVGL、SDRAM、LCD、LoRa 和 ESP8266 的移植与整合。

## 8. 开发注意事项

### 8.1 Cache 与 DMA 一致性

STM32H743 启用数据缓存后，DMA 访问的内存数据可能与 CPU 缓存中的数据不一致。在使用 ADC DMA、UART DMA、SPI DMA、SDMMC 和以太网时，应注意：

- DMA 缓冲区的内存区域；
- Cache Line 对齐；
- DMA 发送前清理数据缓存；
- DMA 接收后使对应缓存失效；
- 必要时使用 MPU 配置非缓存区域。

### 8.2 内存区域划分

STM32H743 的内部 RAM 分布在多个地址区域，不同区域的访问总线和 DMA 支持情况不同。应根据用途合理分配：

- ITCM：存放关键指令；
- DTCM：存放 CPU 高频访问数据；
- AXI SRAM：存放通用数据和较大缓冲区；
- 外部 SDRAM：存放 LVGL 显存和大型图形资源。

需要注意，部分 DMA 控制器不能直接访问 DTCM。

### 8.3 系统时钟配置

STM32H743 的时钟树较复杂。配置主频和外设时钟时，应正确设置：

- 电源供电等级；
- PLL 参数；
- Flash 等待周期；
- AHB 和 APB 分频；
- ADC、USB、SDMMC 和 LTDC 的独立时钟源。

### 8.4 系统可靠性

工业监测系统应增加以下可靠性设计：

- 独立看门狗；
- 通信超时处理；
- 传感器离线检测；
- 数据有效性检查；
- 报警阈值滞回；
- 异常复位原因记录；
- 关键参数掉电保存；
- 云端和本地双重报警。

---

## 9. 总结

STM32H743IIT6 具有较高的处理性能、较大的存储空间和丰富的通信及显示外设，能够同时承担传感器数据采集、实时任务调度、LVGL 图形显示、LoRa 通信、云平台连接、数据存储和报警控制等任务。

因此，本项目选择 STM32H743IIT6 作为主控制器，可以为化工厂有害气体监测系统提供良好的实时性、扩展性和可靠性，并为后续增加更多传感器、通信方式和智能分析功能预留充足的硬件资源。
